1. Сущности
Task:
- id (уникальный идентификатор)
- title (название задачи, обязательно)
- description (описание, опционально)
- completed (статус выполнения, по умолчанию false)
- createdAt (дата создания, устанавливается автоматически)
- category (связь с категорией)

Category
- id
- name (уникальное, обязательно)

2. REST API
Реализовать следующие эндпоинты:

- POST /categories - создать новую категорию
- GET /categories - получить все категории
- POST /tasks - создать задачу (обязательно указать категорию по ID)
- GET /tasks - получить все задачи
- GET /tasks?category={categoryId} - получение всех задач по категории
- GET /tasks?completed=true/false - фильтрация по статусу
- PATCH /tasks/{id}/complete - пометить задачу как выполненную
- DELETE /tasks/{id} - удалить задачу

3. Технические требования
- Java 8
- Spring Boot (любая совместимая версия, например 2.7.x)
- Spring Data JPA + PostgreSQL
- Использовать DTO для входящих/исходящих данных (не возвращать сущности напрямую)
- Настраивать подключение к БД через application.yml
- Корректно обрабатывать ошибки (например, при несуществующей категории — возвращать 204 - NO CONTENT)
